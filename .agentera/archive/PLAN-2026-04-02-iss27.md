# Plan: Narration Voice Warm-Up (ISS-27, Decision 17)

<!-- Level: full | Created: 2026-04-02 | Status: active -->
<!-- Reviewed: 2026-04-02 | Critic issues: 6 found, 5 addressed, 1 dismissed -->

## What
Add a narration voice principle to ecosystem-spec.md and rewrite mechanical narration
in ~5 SKILL.md files. Action narration register with riffable examples, not scripts.

## Why
ISS-26 converged personality and output framing but missed structural narration — mode
announcements ("Returning mode"), transitions ("Starting /[skill]..."), and ad-hoc process
narration still sound like system logs instead of the sharp colleague.

## Constraints
- Structural markers (`─── glyph skill · word ───`) stay unchanged
- Step markers (`── step N/M: verb`) stay unchanged
- Exit signals stay unchanged
- Examples to riff on, not exact scripts (Decision 17: scripts become tics)
- Action narration register: brief, casual, tells you what's happening

## Scope
**In**: ecosystem-spec Section 13, hej narration, visionera/profilera/visualisera mode choice
framing, planera skip message, ad-hoc narration principle for all 11 skills
**Out**: output framing (already done in ISS-26), conversational arc questions (already warm),
resonera scratchpad (functional tool), VISION.md updates (voice principle already captured)
**Deferred**: linter rule for narration voice (would require NLP-level checking)

## Design
ecosystem-spec Section 13 defines the principle and example format that all skills inherit.
Each affected SKILL.md replaces mechanical narration with 2-3 riffable examples per narration
point, using a contrast pair format (mechanical → warm) so the register is learnable. The
remaining 6 skills inherit the principle without specific SKILL.md changes (they have no
mechanical narration points beyond markers).
